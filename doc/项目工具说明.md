# Royce-HomeKit-Mini 项目工具说明

本文档概括仓库中的 Web 功能、API、脚本与相关资源，便于快速查阅。

## 1. Web 应用（主入口）

- **默认首页**：`ROIs_Crew_platform.html`（访问 `/` 或 `/index.html` 会重定向到该文件）。
- **本地端口**：见仓库根目录 `port.txt`（当前为 **8088**）。启动后 `server.py` 会在控制台打印实际监听地址。

## 2. 侧栏功能页（面向用户的工具）

导航与面包屑定义在 `js/nav.js` 中，主要页面如下。

| 页面 ID | 说明 |
|--------|------|
| `bids` | Bids Type Analysis（投标类型分析） |
| `converter` | PDF → Excel Converter（排班 PDF 转表格） |
| `loadsql` | Load SQL into DB（将 SQL 载入数据库） |
| `nbids` | N-Bids Reformat（N-Bids 重排/格式化） |
| `dashboard` | Dashboard |
| `about` | 关于（Us） |
| `client` | 关于 Client |
| `model` | 关于 Model |
| `roadmap` | 关于 Roadmap |
| `dutyswap` | Duty Swap 演示 |
| `dutyswap2` | Duty Swap II |

另有独立 HTML（如 `Client_map.html`、`ROIsCrew_Cycle_v3.html` 等），可通过对应路径由静态文件服务直接访问。

## 3. 后端 API（`server.py`）

| 方法 | 路径 | 用途 |
|------|------|------|
| GET | `/api/livereload` | Server-Sent Events；监听 UI 文件变更并通知浏览器刷新 |
| GET | `/api/sql-files` | SQL 文件列表相关 |
| GET | `/api/whitelist` | 表白名单相关 |
| GET | `/api/analyse-status` | SQL 分析任务状态 |
| POST | `/api/convert` | PDF 排班解析（依赖 `pdfplumber`） |
| POST | `/api/analyse-sql` | SQL 分析 |
| POST | `/api/load-sql` | 将 SQL 加载到数据库 |
| POST | `/api/whitelist-toggle` | 白名单开关/切换 |
| POST | `/api/nbids-reformat` | N-Bids 重格式化 |

`server.py` 顶部注释说明：在配置端口上提供 HTML 静态服务，并提供 PDF 解析等 API。

## 4. 命令行与运维脚本（`scripts/`）

以下为目录中常见脚本类型（具体行为以脚本内注释为准）。

**Shell**

- `run_import.sh`、`import_table.sh`、`load_big_tables.sh`：导入与大批量加载相关。
- `worker.sh`、`monitor.sh`、`watch_progress.sh`：工作进程、监控、进度查看。
- `start_agents.sh`：启动代理/任务相关。

**Python**

- `reload_missing.py`、`fix_failed_tables.py`、`split_big_table.py`：补载、修复失败表、大表拆分等。
- `test_nbids.py`、`test_nbids_browser.py`、`test_ui_pages.py`：测试与页面校验。

其他：`excluded_tables.txt` 等配置/列表文件。

## 5. 其他代码与资料

- **`crew_bids/`**：如 `generate_sql.py` 等，用于 crew bids 相关 SQL 生成与处理。
- **仓库根目录**：如 `split_sql.py` 等独立工具脚本。
- **`Design/`**：SQL、归档脚本、设计说明、Excel/文本等设计资料。
- **`.claude/skills/split-sql.md`**：供 Claude/Cursor 使用的 SQL 拆分技能说明（文档性质，非运行时服务）。

## 6. 开发与服务器约定

仓库根目录 `CLAUDE.md` 约定：修改 `server.py` 后需重启服务，并确保仅有一个 `server.py` 进程在运行（具体命令以该文件为准；路径可能需按本机环境调整）。
